## DescripciГіn del Proceso de Despliegue

### Requisitos Previos
- Azure CLI instalado y autenticado
- Terraform >= 1.0
- Ansible >= 2.9 con colecciГіn `containers.podman`
- Docker o Podman local para construir imГЎgenes
- kubectl

### Paso 1: Infraestructura con Terraform
```bash
cd Terraform
terraform init
terraform apply -auto-approve
```
Crea: Resource Group, VNet, Subnet, VM, NSG, ACR, AKS

### Paso 2: ConstrucciГіn y Push de ImГЎgenes
```bash
./build-push-images.sh
```
Construye y sube al ACR:
- `nginx-https:casopractico2` - NGINX con SSL y autenticaciГіn
- `nodejs-redis:casopractico2` - API Node.js
- `redis:casopractico2` - Redis para persistencia

### Paso 3: ConfiguraciГіn VM con Ansible
```bash
cd Ansible
ansible-playbook -i inventory.ini playbook-podman-https.yml
```
Configura Podman y despliega NGINX HTTPS como servicio systemd

### Paso 4: Despliegue en AKS
```bash
./deploy-aks.sh
```
Despliega en Kubernetes: Namespace, PVC, Redis, Node.js, Services

## DescripciГіn de las Aplicaciones

### AplicaciГіn 1: NGINX HTTPS (Podman en VM)
- **TecnologГӯa**: NGINX en contenedor Podman
- **Puerto**: 443 (HTTPS)
- **Seguridad**: 
  - Certificado X.509 autofirmado (365 dГӯas)
  - AutenticaciГіn bГЎsica htpasswd (usuario: admin, password: admin123)
- **GestiГіn**: Servicio systemd (`podman-nginx-https.service`)
- **Acceso**: `https://<VM_PUBLIC_IP>`

### AplicaciГіn 2: Node.js + Redis (AKS)
- **Frontend**: API Node.js con endpoints REST
  - `GET /items` - Lista todos los items
  - `POST /items` - Guarda item y retorna lista completa
  - `GET /health` - Health check
- **Backend**: Redis con almacenamiento persistente (PVC 1Gi)
- **Persistencia**: Azure Managed Disk via StorageClass `managed-csi`
- **Acceso**: `http://<LOADBALANCER_IP>/items`
